English version
Un modèle de coût pour les configurations clients-serveurs bases de données.
Les systèmes de gestion de Base de Données (SGBD) sont des applications exigeantes en terme de performance. De surcroît, les développements récents des puissances des machines , la fiabilité des moyens de communication, et l'évolution du logiciel changent considérablement les méthodes de conception et de fonctionnement des systèmes de bases de données. Il devient alors, nécessaire d'étudier les performances et les formules de coût des nouvelles techniques utilisées. Dans cette étude nous nous intéressons aux systèmes de bases de données à applications transactionnelles dans un environnement client /serveur et nous étudions leurs performances sous différentes charges de requêtes. Nous portons notre attention sur les performances des sous système du noyau transactionnel. Nous abordons également, l'impact de l'optimiseur sur les performances globales du SGBD. Le soucis principal dans les SGBD centralisés est de diminuer l'impact des E/S sur les performances. L'architecture C/S ajoute une nouvelle contrainte qui est l'optimisation des communications Client-Serveur via le réseau local. Nous analysons alors, les améliorations apportées sur chaque composant du noyau transactionnel pour assurer cette contrainte
