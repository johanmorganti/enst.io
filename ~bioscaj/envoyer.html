send_report:

Syntaxe:

send_report [ fichier | fichier.ps fichier.txt ]

Introduction:

send_report est un utilitaire permettant d'envoyer un document et son abstract pour qui soit accessible à travers la base de données enst-rapports moyennant un client WAIS. Il est possible aussi d'y accéder à partir de la page Mosaic antérieure, en cliquant sur Questionner.

Pour envoyer:

Pour envoyer un document il faut exécuter le script send_report qui se trouve dans le répertoire /usr/local/bin ou ~wais/scripts ou bien /inf/archive/wais/scripts si on travaille sur une machine Solaris.

Pour indiquer le nom du document à envoyer on a deux possibilités:

Si le fichier contenant le document et le fichier contenant l'abstract ont le même nom sauf l'extension, on peut exécuter send_report en indiquant le nom du fichier sans extension.

e.g. si le nom du fichier contenant le document est document.ps et le nom du fichier contenant l'abstract est document.txt on peut exécuter send_report document

Si, par contre, le nom du fichier contenant le document et le nom du fichier contenant l'abstract sont differents, alors il faut exécuter send_report en indiquant le nom du fichier PostScript suivi du nom contenant l'abstract (dans l'ordre indiqué).

e.g. si le nom du fichier contenant le document est document.ps et le nom du fichier contenant l'abstract est document.txt on doit exécuter send_report document.ps document.txt

Observations:

Dans tous les cas le document doit être en format PostScript et le nom du fichier qui le contient doit finir par .ps; de même l'abstract doit être écrit en format texte et le nom du fichier qui le contient doit finir par .txt
