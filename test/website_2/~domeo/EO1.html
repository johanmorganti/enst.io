Concepts et structures de l'Internet

Définition de l'Internet

  • Au sens strict: Réseau d'interconnexion de tous les réseaux qui utilisent la famille des protocoles IP
  • Par extension: Ensemble de tous les réseaux qui coopèrent pour former un réseau virtuel unique pour leurs utilisateurs : l'Internet-Mail, appelé aussi "The Matrix".
  • De manière générale: Un méta-réseau de données global à couverture mondiale.
  • qui utilise l'Internet ?
    • au début : communauté des informaticiens
    • puis : communauté des organismes de R&D et académiques
    • entreprises commerciales offrant des services
    • individus

Les "tuyaux" de l'Internet

  • Supports physiques des réseaux locaux
    • Ethernet 10 Mbits (10 base 5, 10 base 2, 10 base T, 100 base T)
    • Fddi 100 Mbits
    • Token ring
    • Infrarouge
  • Supports physiques longue distance (opérateurs)
    • Lignes spécialisées (Transfix : 9600, 19200, 64K, ..., 2M, 34M)
    • Réseau public à commutation de données (Transpac, IP/X25)
    • RNIS (Numéris : 64K, 2M)
    • Réseau téléphonique (RTC, modem 14,4K V32bis, V42, V42bis)
    • ATM
    • Radio
  • Routeurs (La "glue")
    • routage des paquets et éventuellement filtrage
    • Cisco, Wellfleet, NSC, Proteon, Digital,...
  • Passerelles (vers des réseaux non Internet)
    • trafic Uucp
    • trafic Bitnet
    • trafic X400

Réseaux à commutation de paquets

  • Approche circuit virtuel (X25)
    • métaphore du téléphone
    • un seul paquet d'appel contient les adresses source/destin
    • le paquet d'appel établit un circuit virtuel figé
    • les paquets de données n'ont plus besoin des adresses
    • acquitement de noeuds en noeuds (LAP B), point à point
    • échanges de crédits permanents (contrôle de congestion)
    • maintient d'états (variables) dans le réseau (X75)
    • riche en états (mémorisation des ressources réservées)
    • intelligence dans le réseau
  • Approche datagramme (TCP/IP)
    • métaphore postale
    • paquets indépendants
    • chaque paquet contient leurs adresses source/destination
    • pas d'acquitement interne, contrôle de bout en bout (TCP)
    • maintient de tables de routage par calcul (voisinage)
    • sans état
    • intelligence dans le hôte et l'application

Fonctionnement de l'Internet

Interconnexion point à point

Réseau d'interconnexion = internet (inter-network)

Les réseaux A et B ne peuvent communiquer entre eux que si ils sont interconnectés par un réseau C qu'ils partagent.
Les réseaux A et B possèdent chacun une adresse IP du réseau C.
Le réseau C est un réseau d'interconnexion dit aussi ">internet".

Un internet est un réseau de réseaux de machines capables de communiquer globalement par le biais du protocole IP et ses services. internet d'entreprise : internet privé dans une société internet public : les services Internet offerts au public

DNS Serveur de nom

  • Objectif : adresser les machines par leur nom
    • association d'un nom à un numéro IP unique
    • principe du domainage pour assurer l'unicité des noms
    • administration décentralisée d'autorités de domaines

    • référence au gestionnaire supérieur si échec local
    • utilisation de caches pour améliorer les performances

Telnet Connexion à distance

  • Ouverture d'une session de login distante 

    
    % telnet sirocco.enst.fr
    trying 137.194.160.2...
    Connected to sirocco.enst.fr.
    Escape character is '^]'.

    SunOS UNIX (sirocco)

    login: moi
    password:........

  • Connexion à une application distante
  • syntaxe (port: nom d'un service ou numéro de service):

    telnet hostname port

  • exemples : 
        telnet email.enst.fr mail
    telnet email.enst.fr 25
    telnet solo.enst.fr solo
    telnet metro.jussieu.fr 10000
  • Mode commande (caractère ^] -> prompt: telnet> ) Commandes : ?, open, close, set, mode, status, display, z, quit
  • Fonctionnement : client-serveur

  • Cousin non recommandé : rlogin (proxy)
    • "facility" de Berkeley basée sur la confiance distante du couple hôte-utilisateur
    • fichiers : ~/.rhosts, /etc/hosts.equiv
    • avantage : le "password" ne transite pas sur le réseau
    • inconvénient : modification du .rhost par un tiers

MIME

    • documents PostScript
    • images au format Gif, Tiff, Jpeg
    • son numérisé
    • vidéo encodée Mpeg
    • pointeurs vers des documents externes par ftp
    • type du document : Content-Type: type/sous-type
    • mode de codage : Content-Transfert-Encoding: encodage 
      
    Mime-Version: 01
    Content-Type: text/plain charset=iso-8859-1
    • Metamail (RFC 1521)

      • types de documents, méthodes d'encodage/décodage

POP

  • POP (Post Office Protocol)
    • récupération à distance de la boîte aux lettres
    • ne pas confondre avec "Point of Presence"
    • pop account (compte distant)
    • smtp client côté micro-ordinateur
    • versions de protocoles : POP2, POP3
    • clients : mh, xmh, NeXTMail, Eudora (Mac), Popmail (PC)
    • serveurs : popper (Berkeley), pop3d
    • autre protocole : IMAP (Internet Mail Access Protocol)

Communications interactives

  • Communications interactives entre utilisateurs
    • Talk

      L'écran ou la fenêtre est partagée en 2 zones. Dans la première zone l'émetteur voit ce qu'il tape et est repercuté dans la deuxième zone du récepteur, dans la deuxième zone l'émetteur voit ce que tape son correspondant.

    • Xtalk

      L'émetteur peut ouvrir une fenêtre xtalk sur l'écran de son correspondant qui répondra dans cette fenêtre (nécessite un xhost préalable).

    • Ytalk

      Dialogue à 2 ou plusieurs interlocuteurs. L'écran ou la fenêtre est partitionné autant de fois qu'il y a d'interlocuteurs présents.

    • Irc (Internet Relay Chat)

      Forum continu textuel entre plusieurs utilisateurs. Identification par un "pseudonyme". Sélection d'un canal (channel) de conversation. Réseau virtuel de serveurs IRC permettant de propager les canaux et les "pseudo" actifs.

      -L : liste des serveurs connus et du serveur par défaut

Evolutions et perspectives de l'Internet

Diffusion de groupe - IP-Multicast

  • Réseau virtuel MBONE (expériences multimédia)
    • IP multicast (classe D: 224.xx.yy.zz) : adressage de groupe (protocole IGMP)
    • Portée : TTL (time to live) : LAN, site, région, continent, monde
    • Seuil : threshold (propagation si ttl > threshold)
    • Réseau FMBONE en France
    • Routage provisoire soft par daemons mrouted (tunnels)
    • Technique de "tunneling" : encapsulation IP dans IP
    • Applications multicast
    • sd (Session Directory) : annonce les conférences
    • vat (Visual Audio Tool) : audio-conférence
    • wb (White-Board) : paper-board-conférence
    • nv (Network-Video) : video-conférence
    • telesia : video/audio-conférence
    • mcm (MultiCast Mosaic) : slide-conférence

Sécurité dans l'Internet

  • Filtrage des réseaux et des machines (routeurs)
    • par adresses source/destination
    • par types de services
  • Firewall, garde-barrière
  • Tcp-wrappers
    • tcpd
    • in.gate
  • Authentification
    • authd (RFC 931)
    • identd (RFC 1413)
  • PGP (Pretty Good Privacy)
  • S/Key (One-Time Password System) : password jetable
  • Secure IP (RFC 1038)
  • PEM (Privacy Enhancement Mail)
  • attendre IPv6...

Futur de l'Internet

  • Solution temporaire à l'explosion des adresses
    • CIDR (Classless InterDomain Routing) : classes C aggrégées
  • Solution définitive IPv6 (Toronto - août 1994)
    • IPng (Projet IP next generation) : remplacement de IPv4
    • TUBA (TCP/UDP with Big Addresses, TCP/CLNP)
    • SIPP (Simple IP Plus)
    • IPv6 (futur IP), inspiré de SIPP : 16 octets/128bits
    • Prévision an 2000: plus d'un réseau IP par être humain (15000 réseaux/m2)
  • Privatisation / Commercialisation
    • CIX, GIX, FIX
    • Kiosque
  • Applications
    • Multimédia
    • Impression à distance (remote printing, fax)
    • Mobiles (radio, infrarouge)
    • Télévision Interactive
